I want to do a full UI overhaul of this e-commerce / marketplace app using Tamagui.

**Stack:** React + Create React App (web only)
**App type:** E-commerce / marketplace
**Design vibe:** Clean & minimal, inspired by the "Cactus" theme from tamagui.dev

---

## Cactus theme reference

The Cactus theme on tamagui.dev uses:
- Warm off-whites and sandy/cream backgrounds (not pure white)
- Muted earthy greens as the primary accent
- Warm mid-tone grays for text and borders (not cool/blue-gray)
- Soft, natural feel — nothing harsh or high-contrast
- Dark variant leans warm-dark (dark olive/charcoal, not pure black)

Use this as your visual north star when choosing token values.

---

## Step 1 — Setup

Install and configure Tamagui for Create React App following the tamagui skill.
CRA does not support the Vite plugin — use the Babel preset instead:

- Install `tamagui`, `@tamagui/config`, `@tamagui/babel-plugin`, `@tamagui/themes`
- Add `@tamagui/babel-plugin` to `babel.config.js` or via `craco.config.js`
  (install `@craco/craco` if not already present, and update package.json scripts
  to use `craco` instead of `react-scripts`)
- Create `tamagui.config.ts` with a custom token set (see Step 2)
- Wrap the app root in `TamaguiProvider` in `src/index.tsx` with `defaultTheme="light"`

> Note: Do NOT use `@tamagui/vite-plugin` — it is incompatible with CRA/webpack.

## Step 2 — Cactus-inspired theme

Build a custom theme using `createV5Theme` that captures the Cactus palette:
```ts
const tokens = createTokens({
  color: {
    creamBg:     '#f5f0e8',   // warm off-white page background
    sandSurface: '#ede7d9',   // card / surface background
    cactusLight: '#7a9e6e',   // accent light
    cactusMid:   '#5a7a4a',   // accent primary (buttons, links)
    cactusDark:  '#3d5c30',   // accent pressed / hover
    warmGray100: '#8c8578',   // muted text, placeholders
    warmGray900: '#2e2a22',   // primary text
    darkBg:      '#1e1e16',   // dark mode background
    darkSurface: '#2a2a1e',   // dark mode card surface
  },
})
```

Light theme: warm parchment feel. Dark theme: warm olive/charcoal — not blue-black.

## Step 3 — UI Overhaul

Replace all existing UI with Tamagui components, following both skills throughout.

**Layout**
- Use `XStack`, `YStack` for all layout — no raw `div` flex containers
- Use `ScrollView` for scrollable regions
- Token-based spacing only (`$2`, `$4`, `$true`) — zero hardcoded px values

**Typography**
- Use `H1`–`H4`, `Text`, `Paragraph` from Tamagui
- Keep type scale restrained: 2–3 sizes per page, consistent weight usage

**E-commerce components to overhaul:**
- Product cards → `Card` with image, title, price, and a `Button` CTA
- Navigation / header → `XStack` with logo, nav links, cart icon
- Filters / sidebar → `YStack` with `Checkbox`, `RadioGroup`, `Slider` for price range
- Product detail page → two-column layout with image gallery and purchase panel
- Cart / checkout flow → `Sheet` or `Dialog` for cart drawer, clean form inputs
- Buttons → Tamagui `Button` with cactus accent fill, rounded corners
- Inputs / forms → Tamagui `Input`, `Label`, `Select` — warm-bordered, no harsh outlines

**Interaction states**
- Every interactive element must have `hoverStyle`, `pressStyle`, and `focusStyle`
- Use the cactus accent for focus rings and pressed states

## Step 4 — Polish

Apply the quality bar from the frontend-design skill:
- Palette discipline: warm cream + earthy green accent + warm gray — max 3 colors
- Borders: subtle, warm-toned — never harsh black borders
- Border radius: consistent, `$3` or `$4` for cards, `$2` for inputs and buttons
- Shadows: very subtle warm shadows on cards only
- No pure black (`#000`) anywhere — use `$warmGray900` for text
- Light mode background never pure white — always `$creamBg`
